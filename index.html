Do All Visual Tokens Matter Equally? Object-Evidence Preserving Token Merging for Vision-Language Retrieval

Suhyeong Park1,3, Junha Jung2,3, Jungwoo Park2,3, Jaewoo Kang2,3,*
1The Catholic University of Korea   2Korea University   3AIGEN Sciences Inc.
arXiv preprint 2026
*Corresponding Author
Paper Code arXiv

SaMer compresses post-projector visual tokens with an object-aware feature-spatial merge, then scores with the original MaxSim objective — at inference it is entirely bbox-free.

Abstract

ColPali-style multi-vector retrievers achieve strong vision-language retrieval by storing and scoring many image-side tokens, but this fine-grained representation is expensive: every image contributes hundreds of vectors to storage and to the late-interaction MaxSim score. We ask whether all visual tokens matter equally, and show that they do not — retrieval evidence is concentrated on object regions rather than spread uniformly across the token grid.

We introduce SaMer, an object-evidence preserving token-merging operator that compresses post-projector visual tokens while preserving the original MaxSim retrieval objective. SaMer combines feature-spatial merging, which clusters visual tokens by both feature similarity and spatial proximity, with object-aware assignment, which during training uses Flickr30k-Entities boxes as instance labels to bias a differentiable merge toward object-consistent clusters via an additive inconsistency penalty. Crucially, SaMer uses no auxiliary grounding or bounding-box loss: it freezes the vision encoder and language backbone and adapts only the shared projection layer under the retrieval loss. At inference time SaMer is bbox-free and applies the same feature-spatial merge before MaxSim scoring, cutting the number of stored image-side tokens while keeping retrieval quality intact.

Method

SaMer replaces the dense grid of image-side tokens used by ColPali-style retrievers with a small set of merged region tokens, produced by three coordinated components:

 Feature-Spatial Merging

Clusters visual tokens using a combination of feature similarity and normalized 2D spatial proximity, so that merged regions stay both semantically and spatially coherent.

 Object-Aware Assignment

During training, Flickr30k-Entities boxes provide instance labels that bias the differentiable merge toward object-consistent clusters through an additive inconsistency penalty — with no bbox or grounding loss.

 Projection-Only Adaptation

Freezes the vision encoder and language backbone and trains only the shared projection layer under the retrieval loss, keeping the adaptation lightweight and stable.

At inference SaMer is bbox-free: it applies the same feature-spatial merge before MaxSim scoring. The default configuration merges to K = 64 region tokens with 3 clustering iterations, a spatial weight of 0.1, and an assignment temperature of 0.07, on top of the vidore/colpali-v1.3-hf backbone.

Experimental Setup

We train and evaluate SaMer for text-to-image retrieval on Flickr30k-Entities. Training optimizes a multi-positive text-to-image contrastive loss (temperature 0.07); box annotations are used only to supervise the merge assignment and are never required at inference or cache-building time. Retrieval quality is reported with standard ranking metrics:

  • Recall@1 / @5 / @10 — fraction of queries whose ground-truth image is ranked within the top-k.
  • MRR — mean reciprocal rank of the first relevant image.
  • nDCG — normalized discounted cumulative gain over the ranked list.

The pipeline builds a compressed retrieval cache from the trained projector, then scores queries against merged image-side tokens with MaxSim — storing only K region tokens per image instead of the full token grid. See the code repository for training, caching, and evaluation scripts.
