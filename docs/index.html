Do All Visual Tokens Matter Equally? Object-Evidence Preserving Token Merging for Vision-Language Retrieval

Suhyeong Park1,3, Junha Jung2,3, Jungwoo Park2,3, Jaewoo Kang2,3,*
1The Catholic University of Korea   2Korea University   3AIGEN Sciences Inc.
arXiv preprint 2026
*Corresponding Author
Paper Code arXiv

Abstract

Multi-vector vision-language retrieval preserves fine-grained visual evidence through maximum-similarity late interaction, but dense image-side tokens make storage and scoring expensive. Existing token compression methods reduce this cost, yet they can remove or collapse object- and region-level evidence that future query tokens may need to select. We propose SaMer (Semantic-aware Merging), an object-aware token merging framework that compresses image-side post-projector tokens into \(K\) representative centroids while preserving the original late-interaction interface. SaMer uses object annotations only during training as a merge prior to discourage cross-instance mixing, requires no ground-truth bounding boxes or detectors at inference time, and adapts only the shared projection layer with frozen vision and language backbones. With \(K=64\), SaMer removes more than 93% of image-side tokens and reduces ColPali storage by \(16.09\times\), while improving R@1 on Flickr30K and MSCOCO. These gains arise because object-aware merging preserves query-selectable object evidence that pruning or feature-only pooling can remove or collapse. SaMer also outperforms compression baselines and shows stronger phrase-level grounding, suggesting that efficient multi-vector retrieval depends not only on reducing token count, but on preserving the evidence future query tokens need to select.

Method

SaMer compresses post-projector visual tokens with an object-aware feature-spatial merge, then scores with the original MaxSim objective — at inference it is entirely bbox-free.

SaMer (Semantic-aware Merging) keeps the original late-interaction interface intact and replaces only the image side with a small set of compressed representatives, through three coordinated steps:

Feature-Spatial Merging

Compresses the \(N\) post-projector visual tokens into \(K\) representative centroids by soft assignment over a distance that combines feature similarity and 2D spatial coherence.

Object-Aware Merge Prior

During training only, object annotations add an instance-inconsistency penalty that discourages merging tokens from different object instances — no auxiliary grounding loss, no boxes at inference.

Projection-Only Adaptation

Freezes the vision encoder and language backbone and trains only the shared projection layer, so merged tokens stay compatible with MaxSim scoring.

Multi-vector late interaction

An image \(I\) is encoded into \(N\) visual tokens \(V(I)=\{v_i\}_{i=1}^{N}\) and a query \(q\) into \(M\) tokens \(Q(q)=\{q_j\}_{j=1}^{M}\), all projected into a shared \(D\)-dimensional space. The MaxSim late-interaction score matches each query token to its most similar visual token:

\[ S(q,I) = \frac{1}{M}\sum_{j=1}^{M}\max_{i\in[N]} q_j^\top v_i . \]

Because every query token is compared with every visual token, the number of image-side tokens \(N\) is the bottleneck for both storage and scoring.

Post-projector token merging

Each token \(v_i\) has a normalized coordinate \(p_i\in[0,1]^2\); each representative \(k\) keeps a feature centroid \(\mu_k\) and spatial centroid \(s_k\). SaMer assigns tokens with a distance that combines feature similarity and spatial coherence,

\[ d(i,k) = \underbrace{\left(1-v_i^\top \mu_k\right)}_{\text{feature}} + \gamma\,\underbrace{\left\|p_i-s_k\right\|_2^2}_{\text{spatial}}, \]

where \(\gamma\) controls the spatial term. The distances become soft assignment weights, and each representative is formed as an \(\ell_2\)-normalized weighted centroid:

\[ a_{i,k} = \mathrm{softmax}_{k}\!\left(-\frac{d(i,k)}{\tau_s}\right), \qquad r_k = \mathrm{normalize}_2\!\left(\frac{\sum_{i} a_{i,k}\,v_i}{\sum_{i} a_{i,k}}\right). \]

The compressed set \(R(I)=\{r_k\}_{k=1}^{K}\) is scored with the same rule, now over \(K \ll N\) merged tokens:

\[ S_K(q,I) = \frac{1}{M}\sum_{j=1}^{M}\max_{k\in[K]} q_j^\top r_k . \]

Object-aware merge prior

When object annotations are available at training time, a stop-gradient hard assignment \(c_i=\arg\min_{k} d(i,k)\) estimates each representative's box-label distribution \(P_k(b)\), giving an instance-inconsistency penalty:

\[ P_k(b) = \frac{\sum_{i}\mathbf{1}[c_i=k]\,\mathbf{1}[b_i=b]}{\sum_{i}\mathbf{1}[c_i=k]+\epsilon}, \qquad P_{\mathrm{inst}}(i,k) = 1-P_k(b_i). \]

This penalty is added to the feature-spatial distance so that cross-instance assignments receive lower weight:

\[ a_{i,k} = \mathrm{softmax}_{k}\!\left(-\frac{d(i,k)+P_{\mathrm{inst}}(i,k)}{\tau_s}\right). \]

The penalty is used only during training; at inference SaMer performs bbox-free feature-spatial soft assignment, with the object prior baked into the adapted projection space.

Projection-only adaptation

Only the shared projection layer is trained — merging adds no learnable parameters, yet gradients from the compressed score still flow through the soft-assignment centroids. SaMer is optimized with a multi-positive InfoNCE retrieval loss over compressed MaxSim scores:

\[ \mathcal{L}_{ret} = -\log \frac{\sum_{I^+\in\mathcal{P}(q)} \exp\!\big(S_K(q,I^+)/\tau\big)}{\sum_{I\in\mathcal{B}} \exp\!\big(S_K(q,I)/\tau\big)}, \]

where \(\mathcal{P}(q)\) is the positive set for query \(q\), \(\mathcal{B}\) the batch candidates, and \(\tau\) the temperature. In the main experiments SaMer uses \(K=64\), adapts the projection layer for 3 epochs on Flickr30K-Entities, and keeps both backbones frozen.
